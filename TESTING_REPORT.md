## 🚨 КРИТИЧЕСКИЕ ПРОБЛЕМЫ (ИСПРАВЛЕНЫ)

### ❌ ПРОБЛЕМА #1: Неправильный расчёт частичного закрытия TP
**Статус:** ✅ ИСПРАВЛЕНО

**Описание:**
При закрытии TP уровней, количество рассчитывалось от текущего (уменьшенного) размера позиции, а не от исходного.

**Пример ошибки:**
- Исходная позиция: 100 монет
- TP1 (40%): закрывает 40 монет, остаётся 60
- TP2 (20%): закрывает 20% от 60 = 12 монет ❌ (должно 20!)

**Решение:**
```python
# Добавлено поле initial_quantity
self.open_positions[symbol] = {
    'quantity': quantity,
    'initial_quantity': quantity,  # Сохраняем исходное
    ...
}

# Расчёт от исходного количества
close_qty = pos['initial_quantity'] * tp['size']
```

**Файл:** `main.py:481, 599`

### ❌ ПРОБЛЕМА #2: Отсутствие учёта комиссий
**Статус:** ✅ ИСПРАВЛЕНО

**Описание:**
Модуль `fee_calculator.py` был создан, но не использовался в расчётах PnL.

**Решение:**
```python
# Добавлен учёт комиссий при закрытии TP
if config.ACCOUNT_FOR_FEES:
    position_value = close_qty * current_price
    fees = self.fee_calculator.calculate_total_fees(position_value)
    pnl_usd -= fees
    pnl_percent = (pnl_usd / (close_qty * pos['entry_price'])) * 100
```

**Файл:** `main.py:638-642`

**Комиссии Bybit:**
- Taker: 0.06% (вход + выход = 0.12%)
- Минимальный профит для безубытка: ~0.12%
- TP1 (+4%): чистая прибыль $0.388 (после комиссий $0.012)
