## 🧹 СКРИПТ: cleanup.sh

### Назначение
Автоматическая очистка старых данных для экономии места на диске.

### Что очищается

#### 1. Python Cache
```bash
find . -type d -name "__pycache__" -exec rm -rf {} +
find . -type f -name "*.pyc" -delete
find . -type f -name "*.pyo" -delete
```
**Эффект:** Удаляет скомпилированные Python файлы (~1-5MB)

#### 2. Старые сжатые логи
```bash
find logs/ -name "*.gz" -mtime +30 -delete
find testing/logs/ -name "*.gz" -mtime +30 -delete
```
**Эффект:** Удаляет логи старше 30 дней (~10-50MB)

#### 3. Временные файлы
```bash
find . -type f -name "*.tmp" -delete
find . -type f -name "*.bak" -delete
find . -type f -name "*~" -delete
```
**Эффект:** Удаляет временные файлы (~0-5MB)

#### 4. Старые тестовые логи
```bash
find testing/logs/ -name "*.log" -mtime +60 -delete
find testing/logs/ -name "*.json" -mtime +60 -delete
```
**Эффект:** Удаляет тестовые данные старше 60 дней (~5-20MB)

#### 5. Обрезка основного лога
```bash
if [ "$LOG_SIZE" -gt "$MAX_SIZE" ]; then
    tail -n 5000 "$LOG_FILE" > "$LOG_FILE.tmp"
    mv "$LOG_FILE.tmp" "$LOG_FILE"
fi
```
**Эффект:** Если лог >10MB, оставляет последние 5000 строк

### Результаты очистки

**До очистки:**
```
📊 Размер проекта: ~200-300MB
📝 Логи: ~50-100MB
```

**После очистки:**
```
📊 Размер проекта: ~150-200MB
📝 Логи: ~10-20MB
```

**Экономия:** ~50-100MB

### Безопасность

✅ **Безопасно:**
- Удаляются только кэш и старые логи
- Основные данные не трогаются
- Сделки (trades.json) сохраняются

⚠️ **Внимание:**
- Старые логи удаляются безвозвратно
- Рекомендуется бэкап перед первым запуском

### Запуск

**Вручную:**
```bash
/root/trading_bot_v4_mtf/cleanup.sh
```

**Автоматически:**
```
0 4 * * * /root/trading_bot_v4_mtf/cleanup.sh
```
Каждый день в 4:00 утра

## 🔍 СКРИПТ: system_check.sh

### Назначение
Комплексная проверка состояния бота и системы.

### Проверяемые параметры

#### 1. Статус сервиса
```bash
systemctl status trading-bot.service --no-pager
```
**Показывает:**
- Активен ли сервис
- Время работы (uptime)
- Последние логи systemd

#### 2. Использование ресурсов
```bash
ps -p $BOT_PID -o %cpu,%mem,vsz,rss,cmd
```
**Показывает:**
- CPU: ~0.5-2%
- Memory: ~50-100MB
- VSZ/RSS: Виртуальная/физическая память

**Нормальные значения:**
```
CPU: < 5%
Memory: < 200MB
```

**Тревожные значения:**
```
CPU: > 20% (возможна проблема)
Memory: > 500MB (утечка памяти?)
```

#### 3. Размеры файлов и логов
```bash
du -sh .
du -sh logs/
ls -lh logs/
```
**Показывает:**
- Общий размер проекта
- Размер директории логов
- Список файлов логов с размерами

**Нормальные значения:**
```

Логи: 10-50MB
trading_bot_v4.log: 1-10MB
```

#### 4. Последние записи в логе
```bash
tail -n 10 logs/trading_bot_v4.log
```
**Показывает:**
- Последние 10 строк
- Текущую активность бота
- Возможные ошибки

#### 5. Статистика сделок
```bash
wc -l < logs/trades.json
du -h logs/trades.json
```
**Показывает:**
- Количество записей сделок
- Размер файла сделок

#### 6. Python процессы
```bash
ps aux | grep "[p]ython.*main.py"
```
**Показывает:**
- Все Python процессы бота
- PID, CPU, Memory
- Командная строка запуска

#### 7. Активные подключения
```bash
lsof -p $BOT_PID -a -i
```
**Показывает:**
- Подключения к Bybit API
- Подключения к Telegram API
- Количество активных соединений

**Нормально:**

## 🔄 СКРИПТ: restart_bot.sh

### Назначение
Безопасный перезапуск бота с проверками.

### Процесс перезапуска

#### 1. Проверка .env файла
```bash
if [ ! -f .env ]; then
    echo "❌ Файл .env не найден!"
    exit 1
fi
```

#### 2. Остановка старого процесса
```bash
pkill -f "python.*main.py"
sleep 2

# Принудительная остановка если нужно
if pgrep -f "python.*main.py"; then
    pkill -9 -f "python.*main.py"
fi
```

#### 3. Проверка зависимостей
```bash
if ! python3 -c "import pybit, pandas, numpy, telegram"; then
    pip3 install -r requirements.txt
fi
```

#### 4. Проверка позиций на бирже
```bash
python3 check_positions.py
```
Показывает текущие открытые позиции перед перезапуском.

#### 5. Запуск бота
```bash
nohup python3 main.py > logs/bot_output.log 2>&1 &
BOT_PID=$!
```

#### 6. Проверка успешного запуска
```bash
sleep 3
if ps -p $BOT_PID > /dev/null; then
    echo "✅ Бот успешно запущен!"
else
    echo "❌ Ошибка запуска!"
    cat logs/bot_output.log
fi
```

### Безопасность

✅ **Проверки:**
- Наличие .env файла
- Корректная остановка старого процесса
- Установка зависимостей
- Проверка запуска

⚠️ **Риски:**
- Кратковременная остановка бота (2-5 сек)
- Возможна потеря сигнала во время перезапуска

### Рекомендации

**Когда перезапускать:**
- После обновления кода
- После изменения config.py
- При подозрении на утечку памяти
- Раз в неделю для профилактики

**Когда НЕ перезапускать:**
- Во время активной торговли
- При открытых позициях (проверить сначала)
- В часы пиковой активности рынка

## 🚀 СКРИПТ: deploy.sh

### Назначение
Полное развёртывание бота на новом сервере.

### Процесс развёртывания

#### 1. Остановка старых ботов
```bash
systemctl stop trading-bot.service
systemctl stop trading_bot.service
```

#### 2. Создание директории
```bash
mkdir -p /root/trading_bot_v4_mtf
cd /root/trading_bot_v4_mtf
```

#### 3. Виртуальное окружение
```bash
python3 -m venv venv
source venv/bin/activate
```

#### 4. Установка зависимостей
```bash
pip install --upgrade pip
pip install -r requirements.txt
```

#### 5. Настройка .env
```bash
if [ -f "/root/.env" ]; then
    cp /root/.env /root/trading_bot_v4_mtf/.env
fi
```

#### 6. Создание директорий
```bash
mkdir -p logs
```

#### 7. Установка systemd сервиса
```bash
cp trading-bot.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable trading-bot.service
```

#### 8. Запуск
```bash
systemctl start trading-bot.service
```

#### 9. Проверка
```bash
sleep 3
systemctl status trading-bot.service
```

#### 10. Удаление старых ботов (опционально)
```bash
# Бэкап
tar -czf /root/old_bots_backup.tar.gz /root/bybit_bot

# Удаление
rm -rf /root/bybit_bot
rm -rf /root/bybit_futures_bot
```

### Использование

**Первое развёртывание:**
```bash
# На сервере
cd /root/trading_bot_v4_mtf
./deploy.sh
```

**Обновление:**
```bash
# Загрузить новые файлы через scp
scp -i ~/.ssh/key main.py config.py root@server:/root/trading_bot_v4_mtf/

# Перезапустить
systemctl restart trading-bot.service
```

### Logrotate

#### Конфигурация
```
/root/trading_bot_v4_mtf/logs/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0644 root root
}

/root/trading_bot_v4_mtf/logs/trades.json {
    weekly
    rotate 4
    compress
    missingok
    notifempty
    create 0644 root root
}
```

#### Параметры

**Основные логи:**
- Ротация: ежедневно
- Хранение: 7 дней
- Сжатие: да (gzip)

**Логи сделок:**
- Ротация: еженедельно
- Хранение: 4 недели
- Сжатие: да

#### Проверка
```bash
# Тест конфигурации
logrotate -d /etc/logrotate.d/trading-bot

# Принудительная ротация
logrotate -f /etc/logrotate.d/trading-bot
```
